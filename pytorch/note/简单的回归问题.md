CPU和GPU张量

**CPU张量**存于主存，处理于CPU
**GPU张量**存于显存，处理于GPU

检测GPU设备：
- `cuda.is_available`
- `cuda.device_count`
- `cuda.get_device_name`

转为GPU张量：
- `gpu_tensor = torch.as_tensor(x_train).to(device)`
- to是没有成本的，不管gpu还是cpu都使用

张量的（存储）**位置**
- `type`函数：返回如`torch.cuda.FloatTensor`

GPU张量无法转为numpy数组：
- 使用`cpu`转为cpu张量再转

创建参数

用作**参数/权重**的张量（本质是需要计算其梯度并更新值）
- 可学习参数的张量需要一个**梯度**
- 创建：`requires_grad=True`
- *发送到设备的速度有点慢*


四种创建方式，表现为
1. 不使用GPU才能正常工作
2. 不起作用：会丢失梯度
3. 冗长，但有用
4. 推荐方法

注意：
- 不同设备，不同软件包，即使随机种子相同，伪随机数序列也不同

>在PyTorch中，每个以下画线(`_`)结尾的方法，如上面的 `requires_grad_( )`方法，都是在**原地修改**。也就是说，它们会修改底层变量。

```python
# 1 默认CPU张量
torch.randn(1,requires_grad=True,dtype=torch.float)

# 2 发送到设备(会丢失梯度) 

torch.randn(1,requires_grad=True,dtype=torch.float).to(device)

# 3 先发送后设置
w=torch.randn(1,requires_grad=True,dtype=torch.float).to(device)
w.requires_grad_()

# 4 创建张量并分配设备
w=torch.randn(1,requires_grad=True,dtype=torch.float,device=device)


```
