**未命名命名空间**（也称为**匿名命名空间**）是指没有定义名称的命名空间，如下所示：
```cpp
#include <iostream>

namespace // unnamed namespace
{
    void doSomething() // can only be accessed in this file
    {
        std::cout << "v1\n";
    }
}

int main()
{
    doSomething(); // we can call doSomething() without a namespace prefix

    return 0;
}
```

- 未命名命名空间内的所有标识符都被视为具有内部链接
- 对于函数来说，这实际上与将未命名命名空间中的所有函数定义为静态函数相同
- *未命名命名空间通常不应在头文件中使用*

**内联命名空间**通常用于对内容进行版本管理。
- 与未命名命名空间类似，在内联命名空间中声明的任何内容都被视为父命名空间的一部分。
- 与未命名命名空间不同，*内联命名空间不会影响链接*。

```cpp
#include <iostream>

inline namespace V1 // declare an inline namespace named V1
{
    void doSomething()
    {
        std::cout << "V1\n";
    }
}

namespace V2 // declare a normal namespace named V2
{
    void doSomething()
    {
        std::cout << "V2\n";
    }
}

int main()
{
    V1::doSomething(); // calls the V1 version of doSomething()
    V2::doSomething(); // calls the V2 version of doSomething()

    doSomething(); // calls the inline version of doSomething() (which is V1)

    return 0;
}
```


命名空间可以同时是内联和匿名的：
- 在这种情况下，最好将匿名命名空间嵌套在内联命名空间内。
- 这会产生相同的效果（匿名命名空间内的所有函数默认具有内部链接），但仍能提供一个可用的显式命名空间名称
