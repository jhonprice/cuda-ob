
拷贝构造函数是一种用于使用同一类型的现有对象来初始化新对象的构造函数。在拷贝构造函数执行完毕后，新创建的对象应当是作为初始化器传入的对象的副本。
- 如果你没有为自己的类提供拷贝构造函数，C++ 会为你创建一个公有的**隐式拷贝构造函数**。
	- 默认情况下，隐式拷贝构造函数会执行逐成员初始化。这意味着每个成员都将使用作为初始化器传入的类中对应的成员进行初始化
	- 优先使用隐式拷贝构造函数，除非你有特定的理由需要自己定义一个。
		- 在讨论**动态内存分配**（涉及-- **浅拷贝**与**深拷贝**）时，看到需要重写拷贝构造函数的情况。
- 拷贝构造函数除了执行复制操作外，不应产生任何副作用。


如果一个类没有定义拷贝构造函数，编译器将隐式地为我们生成一个。如果需要，我们也可以使用 `= default` 语法显式请求编译器为我们创建默认的拷贝构造函数


有时我们会遇到不希望某个类的对象可被复制的情况。我们可以通过使用 `= delete` 语法将拷贝构造函数标记为已删除来阻止这种行为：


自定义：
- 拷贝构造函数的参数必须是**左值引用**或 **const 左值引用**。
- 由于拷贝构造函数不应修改其参数，因此推荐使用 const 左值引用。
```cpp
#include <iostream>

class Fraction
{
private:
    int m_numerator{ 0 };
    int m_denominator{ 1 };

public:
    // Default constructor
    Fraction(int numerator=0, int denominator=1)
        : m_numerator{numerator}, m_denominator{denominator}
    {
    }

    // Copy constructor
    Fraction(const Fraction& fraction)
        // Initialize our members using the corresponding member of the parameter
        : m_numerator{ fraction.m_numerator }
        , m_denominator{ fraction.m_denominator }
    {
        std::cout << "Copy constructor called\n"; // just to prove it works
    }

    void print() const
    {
        std::cout << "Fraction(" << m_numerator << ", " << m_denominator << ")\n";
    }
};

int main()
{
    Fraction f { 5, 3 };  // Calls Fraction(int, int) constructor
    Fraction fCopy { f }; // Calls Fraction(const Fraction&) copy constructor

    f.print();
    fCopy.print();

    return 0;
}
```


---

1. 传参：当实参和形参属于相同的类类型时，该复制操作会通过隐式调用拷贝构造函数来完成
2. 返回值：当返回类型与返回值属于相同的类类型时，该临时对象会通过隐式调用复制构造函数进行初始化。
