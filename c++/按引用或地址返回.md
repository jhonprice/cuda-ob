> 返回值的副本会被传递回调用方。如果函数的返回类型是类类型，这可能会造成较高的开销。

**通过引用返回**会返回一个绑定到被返回对象的引用，从而避免创建返回值的副本
- 通过引用返回的对象必须在返回引用的函数作用域结束后仍然存活，否则将导致悬空引用。
- 切勿通过引用返回（非静态）局部变量或临时对象。
- 引用生命周期延长不适用于跨函数边界的情况。
- 避免返回对非常量局部静态变量的引用。

**按地址返回**的工作方式与按引用返回几乎完全相同，区别在于它返回的是指向对象的指针，而非对象的引用。

按地址返回存在与按引用返回相同的主要注意事项：
- 被按地址返回的对象的生命周期必须长于返回该地址的函数的作用域，否则调用者将接收到**悬空指针**

相比按引用返回，按地址返回的主要优势在于：当没有有效对象可返回时，函数可以返回 `nullptr`

按地址返回的主要缺点是，调用者必须在解引用返回值之前记得进行 `nullptr` 检查，否则可能会发生空指针解引用，从而导致未定义行为。

> 除非需要返回“无对象”（使用 `nullptr` ）的能力，否则应优先选择按引用返回而非按地址返回。


示例如下：
```cpp
#include <string>

#include <iostream>

// 按引用返回string的foo函数

// Warning: 这是一个悬空引用的case，返回的引用指向一个局部变量，局部变量在函数结束时会被销毁

[[maybe_unused]] std::string& foo_1() {

    std::string str="abc";

    return str;

}
  

//Warning: 这是悬空索引的另一个case,引用生命周期延长不适用于跨函数边界的情况

[[maybe_unused]]const std::string& foo_2() {

    return "aaa";

}

//Info:通过引用返回的对象必须在返回引用的函数作用域结束后仍然存活，否则将导致悬空引用

std::string& foo_3() {

    static std::string tmp = "abc";

    return tmp;

}
  

int main() {

    std::cout << "hello world" << std::endl;

    const std::string& str = foo_3();

    std::cout << str << std::endl;

    return 0;

}
```
