局部变量默认具有**自动存储期**，这意味着它们在定义时创建，并在退出代码块时销毁。

对局部变量使用 `static` 关键字会将其存储期从**自动存储期**更改为**静态存储期**。
- 这意味着该变量现在在程序开始时创建，并在程序结束时销毁（就像全局变量一样）。
- 因此，即使超出作用域，静态变量仍将保留其值！

当需要一个局部变量在函数调用之间记住其值时，可以使用静态局部变量。
- 初始化你的静态局部变量。静态局部变量仅在代码首次执行时初始化，而非在后续调用中。
- 正如我们使用"g_"作为全局变量的前缀，通常也会使用"s_"作为静态（静态存储期）局部变量的前缀。

**静态局部常量**最适合用于避免每次调用函数时进行昂贵的局部对象初始化。
- const 静态局部变量通常可以放心使用。
- 非 const 静态局部变量通常应避免使用。如果确实需要使用，请确保该变量永远不需要重置，并且不用于改变程序流程。
> 静态局部变量可以被声明为 const（或 constexpr）。const 静态局部变量的一个很好的用途是，当你有一个函数需要使用一个常量值，但创建或初始化该对象的成本很高时（例如，你需要从数据库中读取该值）。如果你使用普通的局部变量，每次函数执行时都会创建并初始化该变量。而使用 const/constexpr 静态局部变量，你可以创建并初始化这个高成本对象一次，然后在每次调用函数时重复使用它。
