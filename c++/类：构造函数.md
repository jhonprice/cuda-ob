编译器会在调用构造函数*之前*为对象设置好内存分配。
随后，构造函数会在未初始化的对象上被调用。
- 如果对于一组初始化器找不到匹配的构造函数，编译器将报错。因此，虽然构造函数本身并不创建对象，但缺少匹配的构造函数会阻止对象的创建。

[[结构体]]聚合类型不允许拥有构造函数——因此，如果为聚合类型添加构造函数，它将不再是聚合类型

编译器会在函数调用时（如有需要）对实参执行**隐式转换**，以匹配参数类型不同的函数定义：构造函数也是如此

构造函数需要能够初始化正在构建的对象，因此构造函数不能是 const。

---
要让构造函数初始化成员，我们需要使用**成员初始化列表**
- 请勿将其与名称相似的“**初始化列表**”混淆，后者用于通过值列表来初始化聚合类型。
```cpp
#include <iostream>

class Foo
{
private:
    int m_x {};
    int m_y {};

public:
    Foo(int x, int y)
        : m_x { x }, m_y { y } // here's our member initialization list
    {
        std::cout << "Foo(" << x << ", " << y << ") constructed\n";
    }

    void print() const
    {
        std::cout << "Foo(" << m_x << ", " << m_y << ")\n";
    }
};

int main()
{
    Foo foo{ 6, 7 };
    foo.print();

    return 0;
}
```

成员初始化列表中的成员变量应按照它们在类中定义的顺序列出。
- 成员始终按照它们在类中定义的顺序进行初始化
- 应尽量避免（如果可能）使用其他成员的值来初始化成员


*一旦成员初始化列表执行完毕，对象即被视为已初始化*。
- 一旦函数体执行完毕，对象即被视为已构造完成。
