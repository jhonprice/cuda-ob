**临时对象**（有时称为**匿名对象**或无名对象）是指没有名称、且仅存在于单个表达式执行期间的对象。
```cpp
#include <iostream>

class IntPair
{
private:
    int m_x{};
    int m_y{};

public:
    IntPair(int x, int y)
        : m_x { x }, m_y { y }
    {}

    int x() const { return m_x; }
    int y() const{ return m_y; }
};

void print(IntPair p)
{
    std::cout << "(" << p.x() << ", " << p.y() << ")\n";
}

int main()
{
    // Case 1: Pass variable
    IntPair p { 3, 4 };
    print(p);

    // Case 2: Construct temporary IntPair and pass to function
    print(IntPair { 5, 6 } );

    // Case 3: Implicitly convert { 7, 8 } to a temporary Intpair and pass to function
    print( { 7, 8 } );

    return 0;
}
```

>括号之所以复杂，是因为它们被过度重载，并用于截然不同的语法结构中。这包括函数调用、对象的直接初始化、临时对象的值初始化、C 风格强制转换、符号/标识符的分组以及变量定义。因此，当您在某些语法中看到括号时……并不总是能明显看出您会得到什么！

我们来看一个稍复杂的例子。考虑语句 `Foo * bar();` 。通过使用或不使用括号，我们可以完全改变该语句的含义：

- `Foo * bar();` （不带额外括号）默认将 `*` 与 `Foo` 分组。 `Foo* bar();` 是一个名为 `bar` 的函数声明，该函数没有参数并返回一个 `Foo*` 。
- `Foo (*bar)();` 显式地将 `*` 与 `bar` 组合在一起。这定义了一个名为 `bar` 的函数指针，它保存一个无参数且返回 `Foo` 的函数的地址。
- `Foo (* bar());` 与 `Foo * bar();` 相同——在这种情况下，括号是多余的。

最后：
- `(Foo *) bar();` 。您可能认为这与 `Foo* bar()` 相同，但这实际上是一个表达式语句，它调用函数 `bar()` ，将返回值进行 C 风格强制转换为类型 `Foo*` ，然后将其丢弃！
C++ 有时确实很奇怪。


---
当函数**按值返回**时，所返回的对象是一个临时对象（使用返回语句中指定的值或对象进行初始化）。

---
在需要将值从一种类型转换为另一种类型且不涉及窄化转换的情况下，我们通常可以选择使用 `static_cast` 或临时对象的显式实例化。
- 作为一个快速经验法则：转换为基本类型时优先使用 `static_cast` ，而转换为类类型时使用列表初始化的临时对象。
