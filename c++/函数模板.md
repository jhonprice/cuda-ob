**模板**定义描述了函数或类的形态。但不同于普通定义（所有类型必须明确指定）
- 在模板中我们可以使用一个或多个占位符类型。
- 占位符类型代表在模板定义时未知、但将在后续（使用模板时）提供的具体类型。


**函数模板**是一种类似函数的定义，用于生成一个或多个重载函数，每个函数具有不同的实际类型集合。这使我们能够创建可以处理多种不同类型的函数。
- 用于生成其他函数的初始函数模板称为**主模板**
- 而从主模板生成的函数称为实**例化函数**

C++支持三种不同类型的模板参数：
- 类型模板参数（模板参数代表一个类型）。
- 非类型模板参数（模板参数代表一个常量表达式值）。
- 模板模板参数（模板参数代表一个模板）。

```cpp
template <typename T> // this is the template parameter declaration defining T as a type template parameter
T max(T x, T y) // this is the function template definition for max<T>
{
    return (x < y) ? y : x;
}
```

> `typename` 和 `class` 关键字没有区别。由于 `class` 关键字较早引入语言，你经常会看到人们使用它。然而，我们更倾向于使用较新的 `typename` 关键字，因为它更清楚地表明类型模板参数可以被任何类型（例如基本类型）替换，而不仅仅是类类型。

> 对于以简单或明显方式使用并代表“任何合理类型”的类型模板参数，使用以 `T` 开头的大写字母（例如 `T` 、 `U` 、 `V` 等）来命名。
> 如果类型模板参数具有非明显的用法或必须满足的特定要求，则需要使用更具描述性的名称（例如 `Allocator` 或 `TAllocator` ）。

类型模板参数可以匹配任何类型，而非模板参数则像普通函数的参数一样工作。
```cpp
// T is a type template parameter
// double is a non-template parameter
// We don't need to provide names for these parameters since they aren't used
template <typename T>
int someFcn(T, double)
{
    return 5;
}

int main()
{
    someFcn(1, 3.4); // matches someFcn(int, double)
    someFcn(1, 3.4f); // matches someFcn(int, double) -- the float is promoted to a double
    someFcn(1.2, 3.4); // matches someFcn(double, double)
    someFcn(1.2f, 3.4); // matches someFcn(float, double)
    someFcn(1.2f, 3.4f); // matches someFcn(float, double) -- the float is promoted to a double

    return 0;
}
```

以告知编译器，禁止使用特定参数实例化函数模板。这通过函数模板特化来实现，它允许我们为特定的模板参数集重载函数模板，同时结合 `= delete` ，这告诉编译器任何使用该函数的地方都应产生编译错误：涉及**模板特化**
```cpp
#include <iostream>
#include <string>

template <typename T>
T addOne(T x)
{
    return x + 1;
}

// Use function template specialization to tell the compiler that addOne(const char*) should emit a compilation error
// const char* will match a string literal
template <>
const char* addOne(const char* x) = delete;

int main()
{
    std::cout << addOne("Hello, world!") << '\n'; // compile error

    return 0;
}
```

> 当**静态局部变量**在函数模板中使用时，从该模板实例化的每个函数都将拥有该静态局部变量的独立版本。如果静态局部变量是常量，这通常不会造成问题。但如果静态局部变量是可修改的，结果可能不如预期。

由于模板类型可以被任何实际类型所替代，模板类型有时被称为**泛型类型**。又因为模板可以独立于特定类型编写，使用模板进行编程有时被称为**泛型编程**。
