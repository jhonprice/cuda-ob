```cpp
#include <iostream>

class Date
{
private:
    int m_year { 2020 };
    int m_month { 10 };
    int m_day { 14 };

public:
    void print()
    {
        std::cout << m_year << '/' << m_month << '/' << m_day << '\n';
    }

    int getYear() const { return m_year; }        // getter for year
    void setYear(int year) { m_year = year; }     // setter for year

    int getMonth() const  { return m_month; }     // getter for month
    void setMonth(int month) { m_month = month; } // setter for month

    int getDay() const { return m_day; }          // getter for day
    void setDay(int day) { m_day = day; }         // setter for day
};

int main()
{
    Date d{};
    d.setYear(2021);
    std::cout << "The year is: " << d.getYear() << '\n';

    return 0;
}
```

Getter 函数应提供对数据的“只读”访问权限。因此，最佳实践是：
- 如果复制成员变量的开销较小，则**按值返回**；
- 如果复制成员变量的开销较大，则**按 const 左值引用返回**。

在创建类时，请考虑以下几点：
- 如果您的类没有不变量，且需要大量访问函数，那么可以考虑使用结构体（其数据成员为公有），并直接提供对成员的访问，而不是使用访问函数。
- 优先实现行为或操作，而非访问函数。例如，与其提供一个 `setAlive(bool)` 设置器，不如实现一个 `kill()` 和一个 `revive()` 函数。
- 仅在公众合理需要获取或设置某个独立成员的值时，才提供访问函数。
