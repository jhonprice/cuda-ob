与普通变量一样，当需要确保类类型对象在创建后不被修改时，通常应将其声明为 **const**（或 **constexpr**）。

一旦 const 类类型对象完成初始化，*任何尝试修改该对象数据成员的操作都将被禁止*，因为这会违反对象的 const 属性。
- 这包括直接修改成员变量（如果它们是 public 的）
- 调用用于设置成员变量值的成员函数。
- 即使并未尝试修改任何成员变量，调用一般成员函数仍然构成了 const 违规。
	- 这是因为成员函数本身未被声明为 const。编译器不允许我们在 const 对象上调用非 const 成员函数。
```cpp
struct Date
{
    int year {};
    int month {};
    int day {};

    void incrementDay()
    {
        ++day;
    }
};

int main()
{
    const Date today { 2020, 10, 14 }; // const

    today.day += 1;        // compile error: can't modify member of const object
    today.incrementDay();  // compile error: can't call member function that modifies member of const object

    return 0;
}
```


**const 成员函数**是一种保证不会修改对象本身，也不会调用任何非 const 成员函数（因为它们可能会修改对象）的成员函数。
- 只需在函数原型中，参数列表之后、函数体之前附加 `const` 关键字
- 对于在类定义外部定义的成员函数， `const` 关键字必须同时用于类定义中的函数声明和类定义外部的函数定义： 涉及 **类与头文件**
- **构造函数**不能声明为 const，因为它们需要初始化对象的成员变量，而这要求对其进行修改
- const 成员函数可以像往常一样修改非成员（例如局部变量和函数参数）并调用非成员函数。const 仅适用于成员。
```cpp
#include <iostream>

struct Date
{
    int year {};
    int month {};
    int day {};

    void print() const // now a const member function
    {
        std::cout << year << '/' << month << '/' << day;
    }
};

int main()
{
    const Date today { 2020, 10, 14 }; // const

    today.print();  // ok: const object can call const member function

    return 0;
}
```

> const 成员函数不得：修改隐式对象，调用非 const 成员函数。
> const 成员函数可以：修改不是隐式对象的对象，调用 const 成员函数，调用非成员函数。


如果一个成员函数不会（也永远不会）修改对象的状态，则应将其声明为 const，以便它既能被 const 对象调用，也能被非 const 对象调用。
- const 成员函数也可以在非常量对象上调用


实例化 const 局部变量是创建 const 对象的一种方式，但更常见的做法是通过常量引用将对象传递给函数，从而获得一个 const 对象。
- 也就是说，不管原来对象是否为const对象，通过常量引用传递，就被当作了const对象
- 以下是有问题的代码：
```cpp
#include <iostream>

struct Date
{
    int year {};
    int month {};
    int day {};

    void print() // non-const
    {
        std::cout << year << '/' << month << '/' << day;
    }
};

void doSomething(const Date& date)
{
    date.print();
}

int main()
{
    Date today { 2020, 10, 14 }; // non-const
    today.print();

    doSomething(today);

    return 0;
}
```


可以通过重载成员函数，使同一个函数同时拥有 const 版本和非 const 版本。这是因为 const 限定符被视为函数签名的一部分，因此仅在 const 属性上有所不同的两个函数会被视为不同的函数。

```cpp
#include <iostream>

struct Something
{
    void print()
    {
        std::cout << "non-const\n";
    }

    void print() const
    {
        std::cout << "const\n";
    }
};

int main()
{
    Something s1{};
    s1.print(); // calls print()

    const Something s2{};
    s2.print(); // calls print() const

    return 0;
}
```
