标识符的链接性决定了该名称的其他声明是否指向同一个对象
- 局部变量具有 `no linkage` 


**全局变量**和**函数标识符**可以具有 
- `internal linkage` 
- `external linkage`

具有**内部链接**的标识符可以在单个翻译单元内被看到和使用，但不能从其他翻译单元访问。
- 这意味着如果两个源文件具有相同名称且具有内部链接的标识符
- 这些标识符将被视为独立的
- 并且不会因重复定义而导致 ODR 违规

具有内部链接的标识符可能对链接器完全不可见。
- 它们可能对链接器可见，但被标记为仅用于特定的翻译单元。

具有内部链接的全局变量有时被称为**内部变量**。
- 要使一个非常量全局变量具有内部链接性，我们使用 `static` 关键字。
*常量和常量表达式全局变量*默认具有内部链接性

> `static` 关键字是存储类说明符的一个例子，它同时设置了名称的链接性和存储期。最常用的 `storage class specifiers` 是 `static` 、 `extern` 和 `mutable` 。术语 `storage class specifier` 主要用于技术文档中。

函数标识符也具有链接性。函数默认具有**外部链接性**（我们将在下一课中详细讨论），但可以通过 `static` 关键字设置为内部链接性

在现代 C++ 中，使用 `static` 关键字为标识符赋予内部链接的做法已逐渐失宠。**未命名命名空间**能够为更广泛的标识符（例如类型标识符）提供内部链接，并且更适用于为多个标识符赋予内部链接

通常有两个原因会将标识符设为内部链接：
- 希望确保某个标识符无法被其他文件访问
- 为了避免命名冲突而过于讲究
- 于任何你明确不希望其他文件访问的标识符，都赋予其内部链接性
