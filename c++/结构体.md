struct和 class
- struct（structure 的缩写）是一种程序定义的数据类型
- 允许将多个变量捆绑到单个类型中。


作为结构体一部分的变量称为**数据成员**（或**成员变量**）

结构体的初始化：
- 数据成员默认也不会初始化
- 结构体类型的变量可以是 const（或 constexpr），正如所有 const 变量一样，它们必须被初始化

C++20 引入了一种新的结构体成员初始化方式，称为**指定初始化器**（designated initializers）。
- 指定初始化器允许您显式定义哪些初始化值对应哪些成员。
- 成员可以使用列表初始化或拷贝初始化进行初始化
- 必须按照它们在结构体中声明的顺序进行初始化，否则将产生警告或错误
- 未指定初始化器的成员将进行值初始化。
```cpp
struct Foo
{
    int a{ };
    int b{ };
    int c{ };
};

int main()
{
    Foo f1{ .a{ 1 }, .c{ 3 } }; // ok: f1.a = 1, f1.b = 0 (value initialized), f1.c = 3
    Foo f2{ .a = 1, .c = 3 };   // ok: f2.a = 1, f2.b = 0 (value initialized), f2.c = 3
    Foo f3{ .b{ 2 }, .a{ 1 } }; // error: initialization order does not match order of declaration in struct

    return 0;
}
```



**聚合数据类型**（简称**聚合类型**）是指任何可以包含多个数据成员的类型。
- 某些聚合类型允许其成员具有不同的类型（例如结构体）
- 另一些则要求所有成员必须属于同一类型（例如数组）

C++ 中，聚合（aggregate）的定义更为狭窄且相当复杂
- C风格数组
- 满足以下条件的类类型（struct、class 或 union）
	- 没有用户声明的构造函数
	- 没有私有或受保护的非静态数据成员
	- 没有虚函数
- `std::array` 也是一个聚合体

聚合类型使用一种称为**聚合初始化**（aggregate initialization）的初始化形式
- 允许我们直接初始化聚合类型的成员。
- 提供一个初始化列表作为初始值设定项，该列表只是一个由逗号分隔的值组成的花括号列表
- 结构体中的每个成员都按照声明顺序进行初始化
```cpp
struct Employee
{
    int id {};
    int age {};
    double wage {};
};

int main()
{
    Employee frank = { 1, 32, 60000.0 }; // copy-list initialization using braced list
    Employee joe { 2, 28, 45000.0 };     // list initialization using braced list (preferred)

    return 0;
}
```

如果聚合体被初始化，但初始化值的数量少于成员的数量，则每个没有显式初始化的成员将按如下方式初始化：
- 如果成员具有默认成员初始化器，则使用该初始化器。
- 否则，该成员将从空初始化列表进行复制初始化。
- 大多数情况下，这将对这些成员执行值初始化（对于类类型，即使存在接受列表的构造函数，也会调用默认构造函数）

> 通常可以使用空的初始化列表来对结构体的所有成员进行值初始化

> 向聚合类型添加新成员时，最安全的做法是将其添加到定义列表的末尾，这样其他成员的初始化器就不会发生移位。


类似于使用**初始化列表**来初始化结构体，你也可以使用初始化列表为结构体赋值

**指定初始化器**也可用于列表赋值

结构体也可以使用另一个相同类型的结构体进行初始化
- 常见于使用返回同类型结构体的函数返回值来初始化结构体时：涉及传递和返回值与结构体

定义一个结构体（或类）类型时，可以在类型定义中为每个成员提供默认初始化值。
- 对于未标记为 `static` 的成员，此过程有时称为**非静态成员初始化**。
- 该初始化值称为**默认成员初始化器**
> 使用默认成员初始化器（或我们将在后面介绍的其他机制），结构体和类即使在没有提供显式初始化器的情况下也能自行初始化
- 列表初始化器中的显式值始终优先于默认成员初始化值。
- 如果聚合体被初始化，但初始化值的数量少于成员数量，则所有剩余成员将进行值初始化。然而，如果为某个成员提供了默认成员初始化器，则将使用该默认成员初始化器代替
> 对于聚合类型，优先使用值初始化（使用空花括号初始化器），而不是默认初始化（不使用花括号）。
- 话虽如此，程序员对类类型使用默认初始化而非值初始化的情况并不少见。
	- 这部分是出于历史原因（因为值初始化直到 C++11 才引入）
	- 部分是因为存在一种特定情况（针对非聚合类型），其中*默认初始化可能比值初始化更高效*：涉及默认构造函数和默认参数
