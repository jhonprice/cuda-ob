通过在声明左值引用时使用 `const` 关键字，我们告诉左值引用将其引用的对象视为常量。这样的引用被称为**指向常量值的左值引用**（有时称为**常量引用**或 **const 引用**）。
- 指向常量的左值引用也可以绑定到可修改的左值。在这种情况下，通过引用访问被引用的对象时，该对象被视为常量（即使底层对象是非常量）
- 除非需要修改被引用的对象，否则优先使用 `lvalue references to const` 而非 `lvalue references to non-const` 。

*指向常量的左值引用同样可以绑定到右值*:
- 当这种情况发生时，会创建一个临时对象并用右值初始化，然后指向常量的引用将绑定到这个**临时对象**

指向常量的左值引用甚至可以绑定到不同类型的值，只要这些值能够隐式转换为引用类型即可
- 如果尝试将 const 左值引用绑定到不同类型的值，编译器将创建一个与引用类型相同的临时对象，使用该值初始化它，然后将引用绑定到该临时对象。

绑定到临时对象的常量引用会延长临时对象的生命周期
- 左值引用只能绑定到可修改的左值。
- 指向 const 的左值引用可以绑定到可修改的左值、不可修改的左值以及右值。这使得它们成为一种更加灵活的引用类型。

**Constexpr 左值引用**,能绑定到具有**静态存储期**的对象（全局对象或静态局部对象）。
- 这是因为编译器*知道静态对象将在内存中的哪个位置实例化*，因此可以将*该地址视为编译时常量*。
- constexpr 引用不能绑定到（非静态）局部变量。这是因为局部变量的地址直到其所在的函数被实际调用时才会确定
- 在定义指向 const 变量的 constexpr 引用时，我们需要同时应用 `constexpr` （应用于引用）和 `const` （应用于被引用的类型）。

```cpp
int main()
{
    static const int s_x { 6 }; // a const int
    [[maybe_unused]] constexpr const int& ref2 { s_x }; // needs both constexpr and const

    return 0;
}
```

例子如下：
```cpp
#include <iostream>

  

int main() {

    // 输出 Hello World

    std::cout << "Hello World" << std::endl;

  

    // 常量左值引用

    const int& a = 10;

    std::cout << "a = " << a << std::endl;

  

    // 常量左值引用绑定变量

    int b = 20;

    const int& c = b;

    std::cout << "c = " << c << std::endl;

    // 常量左值引用延长临时对象生命周期示例

    const std::string& str = std::string("临时字符串对象");

    std::cout << "str = " << str << std::endl;

    // 此处临时对象仍然有效，因为常量左值引用延长了它的生命周期

  
  

    // constexpr 左值引用绑定到静态存储期对象

    static int static_var = 100;  // 静态局部对象

    constexpr const int& ref_static = static_var;  // constexpr 左值引用绑定静态局部对象

    std::cout << "ref_static = " << ref_static << std::endl;

    // 全局对象示例（需要在全局作用域定义，此处仅作说明）

    // 假设有全局变量：int global_var = 200;

    // constexpr const int& ref_global = global_var;  // 可以绑定到全局对象

  
  

    return 0;

}
```
