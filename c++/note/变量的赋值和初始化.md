默认情况下，**赋值**会将 = 运算符右侧的值复制到运算符左侧的变量中。这称为**复制赋值**。


在定义对象时，可以选择性地为对象提供一个初始值。为对象指定初始值的过程称为**初始化**
- 用于初始化对象的语法称为**初始化器**。
- 非正式地，初始值也常被称为“初始化器”。
- 在创建变量时立即进行初始化。

C++中有 5 种常见的初始化形式：
- 自 C++17 起，复制初始化、直接初始化和直接列表初始化在大多数情况下表现一致。
- 但在**类初始化**中会略有区别：涉及类初始化和复制省略
- 另有默认初始化和值初始化

``` C++
int a;         // 默认初始化（无初始化器）

// 传统初始化形式：
int b = 5;     // 复制初始化（等号后的初始值）
int c ( 6 );   // 直接初始化（括号内的初始值）

// 现代初始化形式（推荐使用）：
int d { 7 };   // 直接列表初始化（大括号内为初始值）
int e {};      // 值初始化（空大括号）
```

其他形式的初始化包括：

- 聚合初始化：涉及结构体
- 复制列表初始化
- 引用初始化：涉及左值引用
- 静态初始化、常量初始化和动态初始化：涉及全局变量
- 零初始化

**默认初始化**不执行任何初始化操作，导致变量具有不确定的值（一个不可预测的值，有时被称为“垃圾值”）。

对于某些复杂类型而言，**复制初始化**比其他形式的初始化效率更低，因此在现代 C++中已不再受青睐。
- 然而，C++17 解决了大部分此类问题，复制初始化如今正重新获得支持。
- 每当值被**隐式复制**时，也会使用复制初始化，例如
	- **按值**向函数**传参**
	- **按值**从函数**返回**
	- **按值**捕获**异常**。

直接初始化最初是为了更高效地初始化复杂对象（具有**类类型**的对象）而引入的。
- 它被直接列表初始化所取代。
- 直接列表初始化本身也有一些特殊之处，因此直接初始化在某些情况下再次被使用。
- 当值被**显式转换**为另一种类型时（例如通过 `static_cast` ），也会使用直接初始化。

**列表初始化**有两种形式：
```cpp
int width { 5 };    // 将初始值 5 直接列表初始化到变量 width 中（推荐方式）
int height = { 6 }; // 将初始值 6 复制列表初始化到变量 height 中（很少使用）
```
- 优先使用直接列表初始化或值初始化来初始化变量。
- 列表初始化的主要优势之一是禁止"窄化转换"。这意味着如果你尝试使用变量无法安全存储的值进行列表初始化，编译器必须生成诊断信息（编译错误或警告）来提醒你
- 支持使用**值列表**进行初始化
- 存在一些列表初始化无法按预期工作的情况：涉及`std::vector` 和**列表构造函数**


当使用空花括号初始化变量时，会执行一种特殊的列表初始化形式，称为**值初始化**。

在大多数情况下，值初始化会隐式地将变量初始化为零（或对于给定类型最接近零的值）。在发生归零的情况下，这被称为**零初始化**。


**实例化**是一个专业术语，指变量已被创建（分配内存）+ 初始化（包括默认初始化）。
- 实例化的对象有时被称为**实例**。该术语最常用于类类型对象，但偶尔也适用于其他类型的对象。

C++17 引入了 `[[maybe_unused]]` 属性，它允许我们告知编译器可以接受变量未被使用。编译器将不会为这类变量生成未使用变量的警告。
- 编译器很可能会将这些变量从程序中优化掉，因此它们不会对性能产生影响。

```cpp
#include <iostream>

int main()
{
    [[maybe_unused]] double pi { 3.14159 };  // Don't complain if pi is unused
    [[maybe_unused]] double gravity { 9.8 }; // Don't complain if gravity is unused
    [[maybe_unused]] double phi { 1.61803 }; // Don't complain if phi is unused

    std::cout << pi << '\n';
    std::cout << phi << '\n';

    // The compiler will no longer warn about gravity not being used

    return 0;
}
```
