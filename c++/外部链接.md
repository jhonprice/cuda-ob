**外部链接**的标识符不仅可以在定义它的文件中被看到和使用，还可以在其他代码文件中（通过前向声明）被看到和使用。
- 具有外部链接的标识符是真正“**全局**”的，因为它们可以在程序中的任何地方使用

具有外部链接的标识符对链接器是可见的
- 将一个翻译单元中使用的标识符与另一个翻译单元中的适当定义相连接
- 消除内联标识符的重复，保留一个规范定义：涉及：**内联函数**和**内联变量**

**函数**默认具有外部链接
- 为了调用定义在另一个文件中的函数，使用该函数的其他文件中放置一个函数的前向声明

**非常量全局变量**默认具有外部链接
- 为了使一个全局变量具有外部链接（从而可以被其他文件访问），可以使用 `extern` 关键字来实现


要实际使用在另一个文件中定义的外部全局变量，还必须在任何希望使用该变量的其他文件中放置该全局变量的**前向声明** 。对于变量，创建前向声明也是通过 `extern` 关键字完成的（不带初始化值）
```cpp
#include <iostream>

extern int g_x;       // this extern is a forward declaration of a variable named g_x that is defined somewhere else
extern const int g_y; // this extern is a forward declaration of a const variable named g_y that is defined somewhere else

int main()
{
    std::cout << g_x << ' ' << g_y << '\n'; // prints 2 3

    return 0;
}
```

函数的前向声明不需要使用 `extern` 关键字——编译器能够根据你是否提供函数体来判断你是在定义新函数还是在做前向声明


> 尽管可以通过 `extern` 关键字给 `constexpr` 变量赋予外部链接性，但它们不能作为 `constexpr` 被前向声明。这是因为编译器需要知道 `constexpr` 变量的值（在编译时）

> 仅将 `extern` 用于全局变量的前向声明或 const 全局变量的定义。
> 不要将 `extern` 用于非 const 全局变量的定义（它们隐式地具有 `extern` ）。
