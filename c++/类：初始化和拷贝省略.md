[[变量的赋值和初始化]]
```cpp
#include <iostream>

class Foo
{
public:

    // Default constructor
    Foo()
    {
        std::cout << "Foo()\n";
    }

    // Normal constructor
    Foo(int x)
    {
        std::cout << "Foo(int) " << x << '\n';
    }

    // Copy constructor
    Foo(const Foo&)
    {
        std::cout << "Foo(const Foo&)\n";
    }
};

int main()
{
    // Calls Foo() default constructor
    Foo f1;           // default initialization
    Foo f2{};         // value initialization (preferred)

    // Calls foo(int) normal constructor
    Foo f3 = 3;       // copy initialization (non-explicit constructors only)
    Foo f4(4);        // direct initialization
    Foo f5{ 5 };      // direct list initialization (preferred)
    Foo f6 = { 6 };   // copy list initialization (non-explicit constructors only)

    // Calls foo(const Foo&) copy constructor
    Foo f7 = f3;      // copy initialization
    Foo f8(f3);       // direct initialization
    Foo f9{ f3 };     // direct list initialization (preferred)
    Foo f10 = { f3 }; // copy list initialization

    return 0;
}
```

- 在初始化类类型时，会检查该类的所有构造函数，并使用**重载解析**来确定最佳匹配的构造函数。此过程可能涉及参数的**隐式转换**。
- 在初始化非类类型时，会使用**隐式转换规则**来确定是否存在**隐式转换**。


这几种初始化形式之间存在三个关键差异：
- 列表初始化禁止窄化转换。
- 拷贝初始化仅考虑非 explicit 的构造函数/转换函数。我们将在第 14.16 节——转换构造函数与 explicit 关键字中对此进行讲解。
- 列表初始化会优先匹配列表构造函数，而非其他匹配的构造函数

同样值得注意的是，在某些情况下，某些形式的初始化是被禁止的
- （例如，在构造函数的成员初始化列表中，我们只能使用直接初始化形式，而不能使用拷贝初始化）。
---

**复制省略**是一种编译器优化技术，允许编译器移除不必要的对象复制操作。换句话说，在原本会调用复制构造函数的情况下，编译器可以自由地重写代码，从而完全避免调用复制构造函数。当编译器优化掉对复制构造函数的调用时，我们称该构造函数已被省略。


复制省略（copy elision）不受“仿佛”规则（**as-if rule**）的约束。
- *也就是说，即使复制构造函数具有副作用（例如向控制台打印文本），编译器也允许省略对该复制构造函数的调用！*
- 这正是复制构造函数不应包含除复制操作以外的任何副作用的原因——如果编译器省略了对复制构造函数的调用，这些副作用将不会执行，从而导致程序的可观察行为发生变化！

无需死记编译器何时会或不会进行复制省略。只需知道，这是一种编译器在可行时会执行的优化。如果你预期拷贝构造函数会被调用却未被调用，原因很可能就是复制省略。


在可选的复制省略未执行的情况下，移动语义仍可能允许对象被移动而非复制。
