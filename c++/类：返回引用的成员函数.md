[[按引用或地址返回]]
- 通过引用返回以引用方式传递的函数参数，
- 具有静态存储期的变量（无论是静态局部变量还是全局变量）
通常是安全的，因为这些变量在函数返回后通常不会被销毁。

 例如：

```cpp
// Takes two std::string objects, returns the one that comes first alphabetically
const std::string& firstAlphabetical(const std::string& a, const std::string& b)
{
	return (a < b) ? a : b; // We can use operator< on std::string to determine which comes first alphabetically
}

int main()
{
	std::string hello { "Hello" };
	std::string world { "World" };

	std::cout << firstAlphabetical(hello, world); // either hello or world will be returned by reference

	return 0;
}
```

成员函数还有一个额外的情况需要讨论：通过引用返回数据成员的成员函数
- 返回数据成员的**const左值引用**是安全的。包含该数据成员的隐式对象在函数返回后仍然存在于调用者的作用域中，因此任何返回的引用都将保持有效。
- 返回引用的成员函数应当返回与所返回数据成员类型相同的引用，以避免不必要的类型转换。

对于 getter 函数，使用 `auto` 让编译器根据被返回的成员自动推导返回类型，是一种确保不发生类型转换的有效方法：
```cpp
#include <iostream>
#include <string>

class Employee
{
	std::string m_name{};

public:
	void setName(std::string_view name) { m_name = name; }
	const auto& getName() const { return m_name; } // uses `auto` to deduce return type from m_name
};

int main()
{
	Employee joe{}; // joe exists until end of function
	joe.setName("Joe");

	std::cout << joe.getName(); // returns joe.m_name by reference

	return 0;
}
```

从文档角度来看，使用 `auto` 返回类型会掩盖 getter 函数的实际返回类型:通常更倾向于使用显式的返回类型。

---

由于引用的行为与其所引用的对象完全相同，*因此返回非常量引用的成员函数将提供对该成员的直接访问权限（即使该成员是私有的）。*

**const成员函数**不允许返回对成员的非常量引用。
- 这是合理的——常成员函数不允许修改对象的状态，也不允许调用会修改对象状态的函数。它不应执行任何可能导致对象被修改的操作。
