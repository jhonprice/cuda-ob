如果两个相同标识符以编译器或链接器无法区分的方式出现在同一程序中，编译器或链接器将产生错误。这种错误通常称为**命名冲突**

**作用域区域**是源代码中的一个区域，其中所有声明的标识符都被视为与其他作用域中声明的名称不同（类似于我们类比中的城市）。
- 两个同名的标识符可以在不同的作用域区域中声明，而不会引起命名冲突。
- 在给定的作用域区域内，所有标识符必须是唯一的，否则会导致命名冲突。

**命名空间**提供了另一种类型的作用域区域（称为**命名空间作用域**），允许你在其中声明或定义名称，以消除歧义
- *命名空间只能包含声明和定义*。可执行语句仅允许作为定义的一部分（例如函数定义）。


在类、函数或命名空间内定义的名称，都被视为一个隐式定义的命名空间——**全局命名空间**（有时也称为**全局作用域**）的一部分：涉及**全局变量**
- 其作用域从声明点开始，直至文件结束。
- 通常应避免这样做

当你使用定义在非全局命名空间（例如 `std` 命名空间）内的标识符时，你需要告诉编译器该标识符位于该命名空间：涉及**声明**和**using指令**
- 显式命名空间限定符 `std::`
- 避免在程序顶部或头文件中使用 using 指令（例如 `using namespace std;` ）。它们违背了命名空间被引入的初衷
