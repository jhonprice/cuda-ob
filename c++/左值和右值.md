**表达式**定义为“能够执行以产生单一值的字面量、变量、运算符和函数调用的组合”。
- 表达式的类型必须在编译时确定
- 表达式的值可以在编译时确定（如果表达式是 constexpr）或在运行时确定（如果表达式不是 constexpr）

确定表达式应如何求值以及它们可以在何处使用，C++ 中的所有表达式都有两个属性：**类型**和**值类别**
- **表达式的类型**等同于求值后得到的值、对象或函数的类型
- **值类别**（或子表达式）表明表达式解析为值、函数还是某种对象。

C++11 之前，只有两种可能的值类别： `lvalue` 和 `rvalue`
- 在 C++11 中，为了支持一项名为 `move semantics` 的新特性，增加了三种额外的值类别（ `glvalue` 、 `prvalue` 和 `xvalue` ）。


**左值**是一种表达式，其求值结果是一个**可识别**的对象、函数。
- C++标准中使用了"**可识别**"这一术语，但并未明确定义。
- 可识别的实体（如对象或函数）能够与其他类似实体区分开来（通常通过比较实体的地址实现）
- 可识别的实体可以通过**标识符**、**引用**或**指针**进行访问，其**生命周期**通常长于单个表达式或语句。

**左值**分为两种子类型：
- 可修改左值是指其值可以被修改的左值
- 不可修改左值则指其值无法被修改的左值（因为该左值是 const 或 constexpr 类型）。


**右值**是指不属于左值的表达式。
- 右值表达式会求值为一个值。
- 常见的右值包括：
	- **字面量**（C 风格字符串字面量除外，它们是左值）
	- **按值返回**的函数和运算符的返回值。
- 右值**不可标识**（意味着它们必须立即使用），并且仅存在于它们被使用的**表达式作用域**内。


> 
> 运算符期望其操作数为右值，赋值操作要求其左操作数是一个可修改的左值表达式。

左值可以隐式转换为右值，而右值则不能隐式转换为左值

识别左值和右值表达式的经验法则：
- 左值表达式是指那些求值结果为函数或可识别对象（包括变量）的表达式，这些函数或对象在表达式结束后仍然存在。
- 右值表达式是指那些求值结果为值的表达式，包括字面量和在表达式结束后不再存在的临时对象。
- C++11 中，右值被细分为两种子类型：纯右值和将亡值，因此我们这里讨论的右值是这两类值的总和。
```cpp
#include <iostream>
#include <string>

// T& is an lvalue reference, so this overload will be preferred for lvalues
template <typename T>
constexpr bool is_lvalue(T&)
{
    return true;
}

// T&& is an rvalue reference, so this overload will be preferred for rvalues
template <typename T>
constexpr bool is_lvalue(T&&)
{
    return false;
}

// A helper macro (#expr prints whatever is passed in for expr as text)
#define PRINTVCAT(expr) { std::cout << #expr << " is an " << (is_lvalue(expr) ? "lvalue\n" : "rvalue\n"); }

int getint() { return 5; }

int main()
{
    PRINTVCAT(5);        // rvalue
    PRINTVCAT(getint()); // rvalue
    int x { 5 };
    PRINTVCAT(x);        // lvalue
    PRINTVCAT(std::string {"Hello"}); // rvalue
    PRINTVCAT("Hello");  // lvalue
    PRINTVCAT(++x);      // lvalue
    PRINTVCAT(x++);      // rvalue
}
```
- `operator++` 的结果是左值还是右值，取决于它是作为前缀运算符（返回左值）还是后缀运算符（返回右值）使用！
