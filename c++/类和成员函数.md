在类定义内部，可以在成员变量和成员函数声明之前访问它们。这意味着你可以按任意顺序定义成员变量和成员函数

但，数据成员按照声明顺序进行初始化。如果某个数据成员的初始化访问了另一个尚未声明（因此尚未初始化）的数据成员，则该初始化将导致未定义行为, 因此，通常最好避免在默认成员初始化器中使用其他成员。
```cpp
struct Bad
{
    int m_bad1 { m_data }; // undefined behavior: m_bad1 initialized before m_data
    int m_bad2 { fcn() };  // undefined behavior: m_bad2 initialized before m_data (accessed through fcn())

    int m_data { 5 };
    int fcn() { return m_data; }
};
```


> 如果您的类类型没有数据成员，请优先使用**命名空间**。
> 

在类类型定义内部定义的成员函数会被隐式地视为**内联函数**，因此即使该类类型定义被包含在多个代码文件中，也不会违反**单一定义规则**。
代码如下：
```cpp
// 本示例展示在类类型定义内部定义的成员函数会被隐式地视为内联函数

// 因此即使该类类型定义被包含在多个代码文件中，也不会违反单一定义规则

#include "m_func_odr.h"

// 主程序：演示使用

#include <iostream>
int test();

int main() {

    Counter c1;           // 调用隐式内联构造函数

    c1.increment();       // 调用隐式内联成员函数

    c1.increment();       // 调用隐式内联成员函数

    c1.add(5);            // 调用隐式内联成员函数

    std::cout << "当前值: " << c1.getValue() << std::endl;  // 输出: 7

    c1.reset();           // 调用隐式内联成员函数

    std::cout << "重置后: " << c1.getValue() << std::endl;  // 输出: 0

    test();

    return 0;

}
```

```cpp
#ifndef COUNTER_H

#define COUNTER_H

  

// 类定义头文件 - 可以在多个 .cpp 文件中包含

class Counter {

private:

    int value;

public:

    // 在类定义内部定义的成员函数会被隐式地视为 inline 函数

    Counter() : value(0) {}  // 隐式内联构造函数

    // 隐式内联成员函数

    void increment() {

        ++value;

    }

    // 隐式内联成员函数

    int getValue() const {

        return value;

    }

    // 隐式内联成员函数

    void reset() {

        value = 0;

    }

    // 隐式内联成员函数，带参数

    void add(int n) {

        value += n;

    }

};

  

#endif // COUNTER_H
```
