除非有特殊原因需要改变参数的值（例如函数需要修改参数），否则应优先使用常量引用传递而非非常量引用传递。
```cpp
#include <iostream>

void printRef(const int& y) // y is a const reference
{
    std::cout << y << '\n';
}

int main()
{
    int x { 5 };
    printRef(x);   // ok: x is a modifiable lvalue, y binds to x

    const int z { 5 };
    printRef(z);   // ok: z is a non-modifiable lvalue, y binds to z

    printRef(5);   // ok: 5 is rvalue literal, y binds to temporary int object

    return 0;
}
```


> 使用引用传递时，请确保实参类型与引用类型匹配，否则将导致意外的（且可能代价高昂的）类型转换。

> *经验法则是*：基本类型通过值传递，类类型通过常量引用传递。
> 如果不确定如何处理，请使用常量引用传递，这样更不容易遇到意外行为。
