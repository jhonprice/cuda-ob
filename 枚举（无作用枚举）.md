'**枚举**是一种复合数据类型，其值被限制为一组命名的符号常量（称为**枚举器**）
- C++ 支持两种枚举：*无作用域枚举*和*有作用域枚举*
- 枚举是程序定义类型，在使用每个枚举之前必须对其进行完整定义（前向声明是不够的）

**无作用域枚举**通过 `enum` 关键字定义
- 枚举器隐式为 `constexpr`
- 枚举或枚举类型是程序定义的类型本身
- 枚举器是属于该枚举的特定命名值（例如 `red` ）

枚举类型是**独立的类型**，因此在一个枚举类型中定义的枚举常量不能用于另一个枚举类型的对象
```cpp
enum Pet
{
    cat,
    dog,
    pig,
    whale,
};

enum Color
{
    black,
    red,
    blue,
};
```


无作用域枚举之所以如此命名，是因为它们*将枚举量的名称放入与枚举定义本身相同的作用域中*
- 最好将枚举放在命名作用域（如命名空间或类）中，以避免枚举量污染全局命名空间


默认情况下，第一个枚举量被赋予整数值 `0` ，而后续每个枚举量的值比前一个枚举量大 1
- 除非有充分的理由，否则应避免为枚举量显式赋值
- 如果对枚举进行零初始化（即使用值初始化时发生的情况），即使没有对应值的枚举量，该枚举也会被赋予值 `0` 
- 将表示 0 的枚举器设置为对该枚举最有意义的默认值。如果不存在合适的默认含义，可以考虑添加一个值为 0 的"invalid"或"unknown"枚举器，以便明确记录该状态，并在适当的位置进行显式处理。

尽管枚举存储的是整型值，但它们不被视为整型（而是复合类型）。然而，未限定作用域的枚举可以隐式转换为整型值。由于枚举器是编译期常量，因此这是一种 constexpr 转换
- 用于表示枚举器值的特定整型称为该枚举的**底层类型**（或**基类型**）
- 仅在必要时才指定枚举的基类型
> 由于 `std::int8_t` 和 `std::uint8_t` 通常是字符类型的类型别名，使用其中任一类型作为枚举的基类型很可能会导致枚举量被打印为字符值而非整数值。
```cpp
#include <cstdint>  // for std::int8_t
#include <iostream>

// Use an 8-bit integer as the enum underlying type
enum Color : std::int8_t
{
    black,
    red,
    blue,
};

int main()
{
    Color c{ black };
    std::cout << sizeof(c) << '\n'; // prints 1 (byte)

    return 0;
}
```


编译器会将无作用域枚举隐式转换为整数，但不会将整数隐式转换为无作用域枚举
- 可以使用 `static_cast` 将整数显式转换为无作用域枚举器
- 自 C++17 起，如果未限定作用域的枚举具有显式指定的底层类型，则编译器将允许你使用整型值对该未限定作用域的枚举进行列表初始化


> 由于 C 风格字符串字面量在整个程序运行期间都存在，因此返回一个指向该字符串字面量的 `std::string_view` 是安全的。当 `std::string_view` 被复制回调用者时，其所指向的 C 风格字符串字面量依然有效
